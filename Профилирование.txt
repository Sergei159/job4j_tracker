использовалась программма Java Monitoring & Management console.

1) При добавлении 10 млн элементов в класс MemTracker 
график Heap memory Usage  показывает,что память в heap постепенно увеличивается, но в определенные моменты резко уменьшается -> происходит отчистка мусора 
график Classes сначала возрастает до опредленного значения(кол-ва загруженных JVM классов) и остается константой при операции добавлении элементов.

2) при добавлении 3 мл элементов, при этом в цикле на каждом индексе кратном 1 000 000 производилось удаление 5 000 элементов. График Heap memory Usage возрастал по мере добавления элементов, и был практически не изменен при удалении элементов. При одной полной чистки мусора значения графика уменьшились, затем опять выросли при дальнейшем добавлении элементов.

3) OutOfMemoryError получилось достигнуть, заставив программу добавить 100 млн элементов.
